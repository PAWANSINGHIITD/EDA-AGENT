# 📊 EDA Agent V4

An advanced, AI-powered **Exploratory Data Analysis (EDA)** and **Machine Learning Assistant** built with **Streamlit**, **DuckDB**, and **LangGraph**.

EDA Agent V4 combines a highly interactive analytics dashboard with an autonomous AI co-pilot capable of reasoning, generating code, executing analyses, and self-correcting when errors occur.

---

# ✨ Features

## 🚀 High-Performance Data Ingestion

### Out-of-Core Profiling

Process massive CSV and Parquet datasets using DuckDB without loading the entire dataset into memory.

### Single-Pass Mega Query

Efficiently computes:

* Null counts
* Distinct counts
* Min/Max values
* Distribution statistics
* Skewness metrics
* Column profiling summaries

### Reservoir Sampling

Maintains a representative, memory-safe sample for:

* Visualizations
* LLM analysis
* Sandbox execution

---

## 🧠 AI Co-Pilot & Code Sandbox

### LangGraph Orchestrator

Builds a token-aware context from dataset summaries and routes requests intelligently.

### Self-Correcting Code Execution

The AI can:

1. Generate Python code
2. Execute it safely
3. Capture runtime exceptions
4. Automatically revise and retry execution

### Secure Sandbox Environment

Features include:

* AST allow-list validation
* Isolated subprocess execution
* CPU and memory resource limits
* Restricted code execution policies

# 🛠️ Architecture

The project follows a strict separation-of-concerns design.

## Compute Layer (`src/tools`)

Contains:

* Deterministic mathematical computations
* LLM reasoning modules
* Profiling and analysis logic

Outputs are always structured dictionaries/JSON objects and never UI components.

---

## Registry & Scheduler (`src/tools/registry.py`)

All capabilities are registered with execution costs:

| Cost Type | Description                     |
| --------- | ------------------------------- |
| FREE      | Local deterministic operations  |
| LLM       | Language model calls            |

Selected tasks are executed concurrently using a thread pool.

---

## UI / Rendering Layer (`ui/render.py`)

Responsible for:

* Rendering charts
* Rendering tables
* Dynamic UI generation
* Mapping structured outputs to Streamlit, Plotly, and Altair components

# 📦 Installation

## 1. Clone the Repository

```bash
git clone https://github.com/yourusername/eda-agent.git
cd eda-agent
```

## 2. Create a Virtual Environment

### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

### Windows

```powershell
venv\Scripts\activate
```

## 3. Install Dependencies

```bash
pip install -r requirements.txt
```

## 4. Configure Environment Variables

Copy the example environment file:

```bash
cp .env.example .env
```

Add your Groq API key:

```env
GROQ_API_KEY=your_groq_api_key_here
```

---

# 🚀 Usage

Start the Streamlit application:

```bash
streamlit run ui/app.py
```

# Workflow

## 1. Upload Data

Use the sidebar to upload:

* CSV files
* Parquet files

Explore:

* Schema information
* Statistical summaries
* Data quality flags

generated by the DuckDB profiling engine.

Upload:

* Data dictionaries

* Correlation analysis
* AutoML benchmarks

Generate:

* Statistical EDA
* Distribution plots
* Missing-value analysis

Ask questions such as:

> Plot a histogram of transaction amounts.

> What is the correlation between age and churn?

> Show feature importance for predicting fraud.
